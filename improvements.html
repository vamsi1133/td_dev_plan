Observability

Structured Logging

npm install @nestjs/pino pino-http pino-pretty
// apps/api/src/app.module.ts
import { LoggerModule } from 'nestjs-pino';

@Module({
  imports: [
    LoggerModule.forRoot({
      pinoHttp: {
        level: process.env.NODE_ENV === 'production' ? 'info' : 'debug',
        transport: process.env.NODE_ENV !== 'production'
          ? { target: 'pino-pretty' }
          : undefined,
      },
    }),
  ],
})
  • Include context: userId, rideId, driverId
  • Use request correlation IDs for distributed tracing
  • Log levels: error (failures), warn (recoverable), info (business), debug (dev)

import * as Sentry from '@sentry/node';

});

}

Security

Rate Limiting

npm install @nestjs/throttler
@Module({
  imports: [
    ThrottlerModule.forRoot([
      { name: 'short', ttl: 1000, limit: 3 },      // 3 req/sec
      { name: 'medium', ttl: 60000, limit: 100 },  // 100 req/min
    ]),
  ],
})

Security Headers (Helmet)

npm install helmet
import helmet from 'helmet';
app.use(helmet());

JWT Security Enhancements

  • Token rotation on refresh
  • Invalidate old refresh tokens in Redis
  • Short-lived access tokens (15m)
  • Longer refresh tokens (7d)

Audit Logging

  • Log all admin actions with user ID and payload
  • Track sensitive operations (driver approval, payouts)
  • Use structured logging format for queryability
