Why Firebase Cloud Messaging?

Technology choice justification for TipsyDrive push notifications

Current Selection: Firebase FCM

TipsyDrive Push Notification Needs

Driver Ride Alerts

New ride request notifications to nearby drivers

Ride Status

Driver accepted, arriving, started, completed

Chat Messages

In-app chat between customer and driver

Promotions

Marketing notifications for offers and discounts

Silent Updates

Background data sync for location updates

Topics

City-wide announcements to driver groups

Alternative Options Considered

OneSignal Feature-Rich

Popular push notification platform with advanced features.

Pros

  • Excellent dashboard
  • Advanced segmentation
  • A/B testing built-in
  • Scheduled notifications
  • Generous free tier

Cons for TipsyDrive

  • Paid beyond free tier
  • Another service to manage
  • FCM still used under the hood
  • Extra abstraction layer

Future consideration: OneSignal is great if we need advanced segmentation and marketing automation.

AWS SNS AWS Native

AWS's push notification service integrated with AWS ecosystem.

Pros

  • AWS ecosystem integration
  • Highly scalable
  • Pay-per-use pricing
  • SMS fallback option

Cons for TipsyDrive

  • More complex setup
  • Less React Native support
  • Costs at scale
  • FCM still needed for Android
Expo Notifications Expo Ecosystem

Expo's push notification service for Expo apps.

Pros

  • Simple API (expo-notifications)
  • Works with Expo push tokens
  • Free tier available
  • Easy for Expo apps

Cons for TipsyDrive

  • Adds Expo dependency
  • Less control vs direct FCM
  • Paid beyond free tier
  • Extra abstraction layer
Pusher Beams Pusher Product

Push notification service from Pusher (real-time company).

Pros

  • Clean API
  • Good React Native SDK
  • Device interests
  • Authenticated users

Cons for TipsyDrive

  • Paid service
  • Per-device pricing
  • Smaller ecosystem
  • FCM is free alternative

Bottom Line

For TipsyDrive's push notifications - ride alerts, status updates, and driver broadcasts - Firebase FCM provides:

  • Completely free with unlimited messages
  • Google-grade reliability and infrastructure
  • Excellent React Native support via @react-native-firebase
  • Topic subscriptions for city/zone-based driver alerts
  • Cross-platform (iOS + Android) from single API
  • Data payloads for background processing

Note: For time-critical notifications (new ride requests), we use FCM high-priority with TTL of 60 seconds. For promotional notifications, we use normal priority.
