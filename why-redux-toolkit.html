Why Redux Toolkit?

Technology choice justification for TipsyDrive state management

Current Selection: Redux Toolkit
Official Documentation â†—

What State Redux Toolkit Manages

Auth State

User session, tokens, profile data persisted across app

Active Ride

Current booking status, driver info, fare details

Driver Status

Online/offline, current location, earnings today

UI State

Modals, loading states, navigation history

Notifications

Push notification queue, in-app alerts

Settings

User preferences, app configuration

Note: Server data (rides list, driver list) is managed by TanStack Query, not Redux. Redux handles client-side state only.

Why We Chose Redux Toolkit

Key Advantages

FactorBenefit
Predictable StateSingle source of truth, easy debugging with DevTools
Less BoilerplatecreateSlice reduces Redux code by ~50%
ImmutabilityImmer built-in - write mutable code, get immutable updates
TypeScriptExcellent type inference out of the box
Cross-PlatformSame Redux code works in React Native and React web
Persistenceredux-persist for offline support and session restoration

Code Example

// authSlice.ts
const authSlice = createSlice({
  name: 'auth',
  initialState: { user: null, token: null },
  reducers: {
    setCredentials: (state, action) => {
      state.user = action.payload.user;
      state.token = action.payload.token;
    },
    logout: (state) => {
      state.user = null;
      state.token = null;
    },
  },
});

// Usage in component
const user = useSelector(state => state.auth.user);
dispatch(setCredentials({ user, token }));

Alternative Options Considered

Zustand Lightweight Store

Minimal, unopinionated state management with hooks-first API.

Pros

  • Extremely simple API
  • Tiny bundle size (~1KB)
  • No boilerplate at all
  • Works great with TypeScript
  • No providers needed

Cons for TipsyDrive

  • Less structured for larger teams
  • DevTools not as mature
  • Fewer middleware options
  • Smaller community than Redux
  • Less documentation/tutorials

When to use: Small to medium apps, when Redux feels like overkill, or when bundle size is critical.

MobX Reactive State

Reactive state management using observables and automatic tracking.

Pros

  • Automatic re-renders (no selectors)
  • Mutable syntax feels natural
  • Great for complex derived state
  • Less code than Redux
  • Excellent for real-time updates

Cons for TipsyDrive

  • Magic can be hard to debug
  • Less predictable than Redux
  • Smaller job market
  • Team more familiar with Redux
  • Harder to onboard new developers

When to use: Complex reactive UIs, real-time dashboards, or when team prefers OOP style.

React Context + useReducer Built-in React

React's built-in state management using Context API with useReducer hook.

Pros

  • No external dependencies
  • Built into React
  • Simple for small state
  • Easy to understand

Cons for TipsyDrive

  • Re-renders all consumers on any change
  • No DevTools for debugging
  • No built-in persistence
  • Boilerplate for multiple contexts
  • Performance issues at scale

When to use: Simple apps, theme/locale settings, or when you want zero dependencies.

Jotai / Recoil Atomic State

Atomic state management - each piece of state is an independent atom.

Pros

  • Fine-grained re-renders
  • Easy code-splitting
  • Great for derived state
  • Suspense-friendly
  • Minimal boilerplate

Cons for TipsyDrive

  • Different mental model
  • Less mature ecosystem
  • Harder to debug complex flows
  • Recoil (Meta) has uncertain future
  • Team unfamiliar with atomic pattern

When to use: Apps with many independent state atoms, or when fine-grained reactivity is critical.

Bottom Line

For TipsyDrive's state management - handling auth, active rides, driver status, and UI state across React Native and React web - Redux Toolkit provides:

  • Predictable state with excellent DevTools for debugging
  • Same code works in React Native (mobile) and React (admin)
  • Built-in persistence for offline support (redux-persist)
  • Strong TypeScript support with minimal setup
  • Large community and easy hiring in India

Architecture note: We use Redux Toolkit for client state only. Server state (API data) is managed by TanStack Query, avoiding the common Redux anti-pattern of caching server data in Redux.
