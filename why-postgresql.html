PostGIS Query Example

-- Find drivers within 3km of pickup
SELECT d.*,
  ST_Distance(
    d.location::geography,
    ST_MakePoint(77.5946, 12.9716)::geography
  ) as distance_meters
FROM drivers d
WHERE d.is_online = true
  AND d.status = 'approved'
  AND ST_DWithin(
    d.location::geography,
    ST_MakePoint(77.5946, 12.9716)::geography,
    3000  -- 3km in meters
  )
ORDER BY distance_meters
LIMIT 10;

Alternative Options Considered

MySQL Popular RDBMS

World's most popular open-source database.

Pros

  • Very widely used
  • Simple to operate
  • Good performance
  • AWS RDS support

Cons for TipsyDrive

  • Spatial support is basic
  • No PostGIS equivalent
  • JSONB less powerful
  • Less extensible
MongoDB Document Database

Flexible document store with built-in geospatial.

Pros

  • Flexible schema
  • Built-in geospatial (2dsphere)
  • Horizontal scaling
  • Good for rapid prototyping

Cons for TipsyDrive

  • Weaker relational support
  • Joins are expensive ($lookup)
  • ACID only since v4.0
  • Geospatial less powerful than PostGIS
CockroachDB Distributed SQL

PostgreSQL-compatible distributed database.

Pros

  • Distributed ACID
  • PostgreSQL wire protocol
  • Auto-scaling
  • Survives zone failures

Cons for TipsyDrive

  • Limited PostGIS support
  • Higher latency than single-node
  • More complex operations
  • Overkill for MVP scale

Future consideration: When scaling to multiple cities with high write throughput, CockroachDB or Aurora PostgreSQL could be evaluated.

DynamoDB AWS Serverless

AWS managed NoSQL with auto-scaling.

Pros

  • Fully serverless
  • Auto-scaling to infinity
  • Single-digit ms latency
  • Zero maintenance

Cons for TipsyDrive

  • No geospatial support
  • No joins or relations
  • Complex access patterns
  • Vendor lock-in

Bottom Line

For TipsyDrive's database - requiring geospatial queries, complex relations, and ACID transactions - PostgreSQL + PostGIS provides:

  • Industry-standard geospatial with PostGIS (ST_DWithin, ST_Distance)
  • Proper relational modeling for users, drivers, rides, payments
  • ACID transactions for payment processing
  • JSONB for flexible metadata storage
  • Excellent AWS RDS support with Multi-AZ for reliability
