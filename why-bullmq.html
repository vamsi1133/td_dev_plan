Code Example

// Define queue
@Processor('notifications')
class NotificationProcessor {
  @Process('push')
  async handlePush(job: Job) {
    await this.fcm.send({
      token: job.data.deviceToken,
      notification: job.data.notification,
    });
  }
}

// Add job with retry
await this.notificationQueue.add('push', {
  deviceToken: user.fcmToken,
  notification: { title: 'Driver arriving!' }
}, {
  attempts: 3,
  backoff: { type: 'exponential', delay: 1000 }
});

Alternative Options Considered

Agenda MongoDB-Based

Job scheduling library using MongoDB as storage.

Pros

  • MongoDB persistence
  • Good for cron-like jobs
  • Agendash UI available
  • Mature library

Cons for TipsyDrive

  • We use PostgreSQL, not MongoDB
  • Would add new DB dependency
  • Slower than Redis-based
  • Less active development
AWS SQS + Lambda Serverless

AWS managed message queue with Lambda processing.

Pros

  • Fully managed, no infrastructure
  • Auto-scaling
  • Very reliable
  • Pay-per-message

Cons for TipsyDrive

  • Lambda cold starts add latency
  • Complex for NestJS integration
  • Harder to test locally
  • Vendor lock-in

Future consideration: May consider SQS for high-throughput event processing at scale.

RabbitMQ Message Broker

Full-featured AMQP message broker.

Pros

  • Very powerful routing
  • Multiple exchange types
  • Reliable message delivery
  • Management UI built-in

Cons for TipsyDrive

  • Another service to manage
  • More complex than needed
  • Overkill for our use case
  • Redis already available
Bee-Queue Lightweight Redis Queue

Simple, fast Redis queue (predecessor concept to BullMQ).

Pros

  • Very fast
  • Simple API
  • Low overhead
  • Redis-based

Cons for TipsyDrive

  • Limited features vs BullMQ
  • No official NestJS module
  • Less active maintenance
  • TypeScript support limited

Bottom Line

For TipsyDrive's background jobs - notifications, SMS, payment processing, and scheduled tasks - BullMQ provides:

  • Uses existing Redis - no additional infrastructure
  • First-class NestJS integration via @nestjs/bullmq
  • Excellent TypeScript support with full type safety
  • Delayed jobs for scheduled driver payouts
  • Automatic retries with exponential backoff
  • Bull Board dashboard for job monitoring
